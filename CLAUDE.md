# Shift-management — プロジェクトメモ

シフト管理アプリ（Next.js + Supabase）。公式LINE連携（ログイン・通知・打刻・
入口スマートロック）を備える。

## UI設計ルール（重要）

**画面はできるだけスクロールせずに操作・把握できるよう、コンパクトで分かりやすく設計すること。**

- 1画面（ファーストビュー）で全体が見渡せる情報密度を優先する。冗長な余白・巨大カード・
  間延びしたリストは避ける。
- 情報量が多い場合は、表・タイムライン・アコーディオン・サマリ表示などで「畳んで」見せる。
  （例：プレオープン予約は枠アコーディオン＋空き状況テーブル）
- 重要な情報（埋まり具合・状態・件数など）は一目で分かる位置（行の右端・見出し横）に出す。
- モバイル（公式LINE/LIFF想定）でも縦スクロールが最小になるレイアウトを心がける。

## ドキュメント更新ルール（重要）

**仕様・機能に変更を加えたら、対応する `docs/*.md` も必ず同じコミットで更新すること。**

- 新機能を追加した → 該当の docs を更新、無ければ新規 MD を作る。
- 既存の挙動・環境変数・API・権限モデルを変えた → 関連 docs を直す。
- ドキュメントとコードが食い違ったままにしない（コードが正、docs を追従させる）。

### docs 一覧（どの変更がどの MD に対応するか）

| ドキュメント | 範囲 |
|---|---|
| `docs/LINE_INTEGRATION.md` | LINEログイン / 通知（Messaging API）の設計 |
| `docs/SHIFT_OFFER.md` | 出勤打診エンジン（休み承認→早番/遅番の欠員を自動打診） |
| `docs/PERSONAL_BLACKOUTS.md` | 個別予定の不可時間（タイムツリー画像取り込み→生成で回避） |
| `docs/PREOPEN_RESERVATION.md` | プレオープン簡易予約（モデル客・スタッフ各自・1枠4名） |
| `docs/SMART_LOCK.md` | 入口スマートロック（セサミ）＋ 公式LINEのリッチメニュー |
| `docs/DEPLOY_AND_DOMAIN.md` | デプロイ・独自ドメイン設定 |
| `docs/design-brief.md` | デザイン方針 |

新しい領域の機能を足すときは、上表に行を足したうえで MD を新設する。
