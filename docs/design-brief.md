# シフト管理アプリ デザイン指示文（Claude Design 用）

> 各画面ごとに、該当セクションをコピーして Claude Design に貼り付けてください。
> 冒頭の「共通トンマナ」は毎回先頭に付けると一貫性が出ます。

---

## 0. 共通トンマナ（全画面共通・毎回先頭に貼る）

```
# プロダクト
「全力ストレッチ岐阜長良店」のスタッフ向けシフト管理Webアプリ。
オーナー（管理者）とスタッフが使う。PC・スマホ両対応のレスポンシブ。

# デザインのトンマナ
- ミニマルでシックな印象。余白を大きく取り、情報密度は低め。
- 装飾を排し、タイポグラフィと余白で構成する「エディトリアル（雑誌的）」なレイアウト。
- 配色：背景はオフホワイト/ごく薄いグレー（#F4F3F1 系）。文字はほぼ黒（#1A1A1A）。
  罫線は極細のライトグレー（1px, #E0DED9）。アクセントカラーは控えめに1色だけ
  （くすんだ赤系 #C2402A をリンク・選択状態・重要アクションのみに使用）。
- 大胆な見出し：英字の大きなサンセリフ（軽め〜レギュラーウェイト）を効かせる。
  日本語は細めゴシックで小さく添える。英字ラベルは大文字・字間広め（letter-spacing大）。
- 角丸は最小限（0〜4px）、影は使わない。区切りは「線」で表現する。
- ボタンは塗りつぶしを多用せず、テキスト＋下線や細い枠線を基本にする。
  重要な主アクションだけ塗り（アクセント色）。
- アイコンは細い線（ストロークアイコン）。矢印は「↗ ↘ →」など細い記号を使う。
- 全体に静かで知的、余白の効いた高級感。賑やかさ・カラフルさは避ける。
```

## 1. ログイン画面（/login）

```
画面：ログイン
役割：オーナー・スタッフ共通の入口。
構成：
- 画面中央に最小限のフォーム。周囲は大きな余白。
- 上部に店名ロゴ（テキストロゴ「全力ストレッチ岐阜長良店」、英字字間広め）。
- 入力欄2つ：メールアドレス、パスワード（ラベルは小さめ、枠は下線または極細枠）。
- ログインボタン（唯一の塗りボタン＝アクセント色、横幅いっぱい）。
- エラー時：フォーム上に小さな赤テキストで「メールアドレスまたはパスワードが正しくありません。」
機能：メール＋パスワードでログイン。成功で権限に応じた画面へ遷移。
雰囲気：白い静かな1カラム。中央寄せ。装飾なし。
```

## 3. オーナー：ダッシュボード（/admin）

```
画面：ダッシュボード（オーナーのトップ）
構成：
- 大きな見出し「Dashboard」（英字大）＋小さく日本語「ダッシュボード」。
- 上段：サマリーカード3枚を横並び（スマホは縦積み）。各カードはラベル小＋数値特大。
  ① 登録スタッフ（◯名） ② 未対応の休み希望（◯件） ③ シフト作成への導線。
  カードは塗らず、極細枠 or 罫線区切り。数字を主役にする。
- 中段：直近月のシフトカレンダー（後述のカレンダーUIを埋め込み）。右上に「編集 →」リンク。
- 下段：最近のシフト期間リスト（年月＋ステータス「下書き/公開/確定」をタグ表示、行は罫線区切り）。
機能：各カードから各管理画面へ遷移。カレンダーで当月の勤務を俯瞰。
雰囲気：数値とカレンダーが主役。余白広め、罫線でセクション分け。
```

## 4. カレンダーUI（ダッシュボード・シフト確認・シフト編集で共通利用）

```
画面：シフトカレンダー（コンポーネント）
構成：
- 上部に「スタッフで絞り込む」アコーディオン（開くと、スタッフ名チップを複数選択。
  各チップは本人の表示色の丸＋氏名。選択中は塗り、「全員」でリセット）。
- ビュー切替タブ：「月 / 週」（選択中はアクセント色）。右側に期間ラベルと ← → の送りボタン。
- 月ビュー：7列のカレンダーグリッド。日曜赤・土曜青の薄い色分け。
  各日セルにその日の勤務を「● 氏名 10:00–19:00」の小さな行で最大5件、超過は「＋N」。
  日付クリックでその週の週ビューへ。
- 週ビュー：1カラムで日付ごとのセクションを縦に並べる。各セクション見出しに
  「6/20（土）」＋右に人数「3名」、本日は「本日」バッジ。
  その下に勤務を縦積み（● 表示色 / 氏名 / 右寄せで時間帯）。勤務なしは「シフトなし」。
機能：月/週切替、スタッフ絞り込み、自分の勤務をハイライト（スタッフ画面時）。
雰囲気：グリッドの罫線は極細グレー。色は各スタッフの識別色を薄く使うのみ。
```

## 5. オーナー：スタッフ管理（/admin/staff）

```
画面：スタッフ管理
構成：
- 見出し「Staff」＋日本語小見出し。
- 「＋ 新規スタッフ登録」アコーディオン（普段は折りたたみ。開くと登録フォーム）。
  フォーム項目：氏名 / メール / パスワード / 権限（オーナー・スタッフ）/ 雇用形態 /
  電話 / 時給 / 週の最低時間 / 週の最大時間 / 表示色（カラーピッカー）。登録ボタンは塗り。
- 「登録済みスタッフ（◯名）」一覧：
  - PC：テーブル（氏名＋色丸 / 権限 / ログインID / 初期PW / 雇用形態 / 週時間 / 状態 / 編集）。
  - スマホ：1人1枚のカード（横スクロールなし）。氏名＋編集リンクを上段、
    権限・ログインID・初期PW・週時間・状態を定義リスト風に縦並び。
状態は「稼働中（薄緑）/ 停止（薄グレー）」のタグ。
機能：新規登録、各行から詳細編集へ。
雰囲気：テーブルは罫線のみで区切る。ゼブラや影は使わない。
```

## 6. オーナー：スタッフ詳細（/admin/staff/[id]）

```
画面：スタッフ詳細・編集
構成：パンくず（スタッフ管理 / 氏名）＋氏名大見出し。以下カードを縦に並べる：
- プロフィール：権限・雇用形態・電話・時給・週の最低/最大時間を定義リスト表示。
- ログイン情報：ログインID（メール）とパスワードの入力欄＋「ログイン情報を更新」ボタン。
  注記「空欄ならパスワードは変更しません」。
- 希望シフト（週次）：曜日×時間帯のエディタ。曜日ごとに「希望(優先)/可/不可」と時間帯を登録。
- 固定シフト（週次の確定パターン）：曜日ごとに固定の出勤時間を追加/削除できる。
  日〜土の7カラム（スマホは折返し）、各曜日に登録済みパターンを小さく列挙。
機能：プロフィール確認、ログイン情報更新、希望/固定シフトの編集。
雰囲気：カードは罫線区切りでセクション分け。フォームは下線入力主体。
```

## 8. オーナー：シフト編集（/admin/shifts/[id]）

```
画面：月次シフトの生成・編集・公開
構成：
- 見出しに対象年月。ステータス（下書き/公開/確定）をタグ表示。
- 生成パネル（カード）：横並びの3ボタン
  ①「📌 固定シフトを展開」（主アクション＝塗り。固定パターンを当月へ一括反映、希望休は自動除外）
  ②「🤖 ソルバーで自動生成」（副次・枠線ボタン）
  ③「🧠 Claudeで生成（店舗ルール参照）」（副次・枠線ボタン）
  実行後、結果メッセージを小さく表示（成功＝控えめな緑、失敗＝赤）。
- その下にカレンダーUI（月/週切替）でシフト表を表示・確認。
- 公開/確定アクション（状態を進めるボタン）。
機能：固定シフト展開・AI生成・確認・公開・確定。
雰囲気：操作ボタンは「主1＋副2」で視覚的に主従をつける。賑やかにしない。
```

## 補足（デザイナーへの指針）

- 参考イメージは「tacto inc.」のような、英字大見出し＋大きな余白＋極細罫線＋
  モノトーン基調＋ごく一部だけアクセント色、というエディトリアルなミニマル表現。
- 数字・見出し・余白で「間」を作り、UI部品は最小限にする。
- スマホでは横スクロールを発生させない（テーブルはカード化）。
- アクセント色は「リンク・選択・主アクション」だけに限定し、使いすぎない。
